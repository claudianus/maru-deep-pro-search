내장 웹 검색의 6가지 치명적 결함 6 Fatal Flaws of Built-in Web Search

편리해 보이지만, 실제로 의존하면 에이전트가 환각에 빠집니다 Convenient in theory. Dangerous in production.

단일 엔진 의존

Single Engine

단일 검색 엔진에만 의존하면, 차단·장애 시 전체 조사가 중단됩니다. 엔진 간 전환 없이 단일 지점 장애(SPoF)가 됩니다.

Relying on one engine means a block or outage stops research cold—no failover, single point of failure.

랭킹 부재

No Ranking

엔진이 반환한 순위를 재평가하지 않습니다. 관련도·신뢰도·최신성을 반영한 재랭킹 단계가 없습니다.

Raw engine order. No BM25, no authority scoring, no freshness filtering.

인용 환각

Citation Hallucination

출처를 허위로 만들거나 생략하기 쉽습니다. 응답 본문과 1:1로 대응하는 네이티브 인용 ID가 없으면 검증이 어렵습니다.

Makes up sources or ignores them. No native citation IDs means zero traceability.

얕은 콘텐츠

Shallow Fetch

스니펫만 발췌해 종료합니다. API 문서, 버전 표, 코드 예시를 놓치고 저신호 블로그에 의존하기 쉽습니다.

Grabs a snippet and stops. Misses API docs, version tables, code examples entirely.

방어 없음

Zero Defense

임의의 웹 페이지를 필터 없이 가져옵니다. 프롬프트 인젝션, 제로폭 문자, 악성 콘텐츠에 무방비입니다.

Fetches arbitrary pages with no sanitization. Prompt injection, zero-width chars, malicious content all pass through.

강제 불가

Not Enforced

"검색할 수 있다"일 뿐, 아무것도 강제하지 않습니다. 에이전트는 여전히 낡은 학습 데이터를 기본으로 사용합니다.

The agent *can* search, but nothing forces it. It still defaults to stale training data every single time.

프로덕션급 검색 MCP Supreme Research Platform

검색 도구가 아닙니다. AI가 낡은 사전지식만 믿고 코딩하지 못하도록 막습니다. Not a standalone search tool. A search MCP server with harness setup tools.

9엔진 자동 폴오버

9-Engine Failover

9개 엔진을 순차·정책 기반으로 시도하며, 차단·장애·품질 저하 시 자동으로 다음 엔진으로 전환합니다.

Nine engines with automatic failover. One goes down, the next picks up instantly. Resilient by design.

DuckDuckGo Bing Google Yahoo Ecosia Baidu Startpage Naver

지능형 재랭킹

Smarter ordering

BM25·RRF·의미 유사도·커버리지·접근성·노이즈 패널티를 결합해 상위 결과에 고신호 문서가 오도록 정렬합니다.

BM25 + RRF + semantic similarity + coverage/access/noise reranking. Best sources rise to the top.

네이티브 인용

Native Citations

주장 단위로 `[1]`, `[2]` 형태의 추적 가능한 인용을 붙이기 쉬운 응답 형식을 유지합니다.

Every claim gets `[1]`, `[2]`, `[3]`. Sources are real, traceable, and embedded directly in responses.

딥 리서치 파이프라인

Deep Research Pipeline

쿼리 확장 → SERP 단위 RRF → Insights·Clusters·Blueprint 출력. 기본 30소스 · 7서브쿼리.

Query expansion → SERP-level RRF → Trace/Insights/Clusters/Blueprint. Default 30 sources · 7 subqueries.

프롬프트 인젝션 방어

Prompt Injection Defense

수집 본문에서 제로폭 문자·채팅 유도 구문 등을 정제하고, 외부 콘텐츠는 기본적으로 비신뢰 입력으로 취급합니다.

Sanitizes fetched content: strips zero-width chars, neutralizes chat tokens, flags suspicious patterns. Treats all external content as untrusted.

리서치 우선 강제

Search-before-code guardrails

설치 시 에이전트 설정에 리서치 우선 정책을 주입합니다. 단순 권고 문구가 아니라 서버·세션·규칙 파일이 함께 동작합니다.

Install seeds cite-before-edit policies into agent configs (e.g. `.cursor/rules/*.md`, `CLAUDE.md`, `.kimi/settings.json`). Server, session gate, and files move together—not a lone reminder line.

3계층 강제 아키텍처 3-Layer Enforcement

"검색해주세요"가 아닌, 기술적으로 검색을 강제합니다 Not "please search". Technically enforced.

1

MCP 프롬프트 주입

MCP Prompt Injection

서버가 에이전트에 리서치 우선 안내를 프롬프트로 제공하며, 번들된 대형 리서치 패스가 기본 진입 경로가 됩니다.

Bundled MCP prompts bias the agent toward a full web pass before heavy codegen tools. That bundled pass stays the default entry point.

2

세션 게이트

Session Gate

현재 세션에서 웹 리서치가 수행되지 않았다면 자동 코드 초안 생성을 차단하고, 근거 수집부터 진행하도록 유도합니다.

Auto code drafts stay off until a research pass happened in this chat.

3

에이전트 규칙 주입

Agent Rule Injection

Cursor·Claude 등 에이전트별 규칙 파일에 동일한 리서치 우선 정책을 자동 삽입할 수 있습니다.

Setup CLI writes the same search-and-cite guardrails into each agent’s config files: `.cursor/rules/*.md`, `CLAUDE.md`, `.kimi/settings.json`, and friends.

전체 목록은 README를 참고하세요. 여기서는 일상적으로 자주 쓰는 변수만 요약합니다.

Full table in README — high-signal subset for daily use.

MARU_SEARCH_ENGINE

기본 SERP 엔진. 차단 먹으면 bing·google 등으로 교체.

Default SERP engine; switch to bing / google when blocked.

MARU_DEEP_MAX_SOURCES

기본 30. 빠른 작업은 10–15.

Default 30; use 10–15 for fast tasks.

MARU_DEEP_MAX_SUBQUERIES

쿼리 확장 상한. 기본 7.

Query expansion cap. Default 7.

MARU_SERP_PER_ENGINE_CAP

엔진당 SERP 파싱 상한. 기본 50.

Per-engine SERP cap. Default 50.

MARU_ENABLE_STARTPAGE

1 → Playwright Startpage를 자동 추천에 포함.

1 includes Playwright Startpage in auto-recommend.

MARU_WRAPPER_TIER

tiered(경량) 또는 full.

tiered (light) or full SERP wrappers.

MARU_SEARCH_MAX_RESULTS

web_search 기본 10. 좁은 탐색은 5–8.

web_search default 10; 5–8 for narrow tasks.

MARU_SEARCH_TIMEOUT · FETCH_HTTP · DEEP_RESEARCH · ANSWER · AUTO_FETCH

접미사 _TIMEOUT — SERP / 페이지 fetch / 딥리서치 / answer / auto_fetch 각각 분리.

Suffix _TIMEOUT — SERP, page fetch, deep pipeline, answer, nested auto_fetch.

MARU_STRICT_QUERY

0 → 거절 끔(보정만). 기본은 엄격 모드.

0 disables rejection; optimization stays on.

MARU_EMBEDDING_MODEL

시맨틱 랭킹 Hugging Face 모델 (기본 ibm-granite/granite-embedding-97m-multilingual-r2).

Hugging Face model for semantic ranking (default ibm-granite/granite-embedding-97m-multilingual-r2).

MARU_DEBUG

1이면 MCP 서버 로그 DEBUG.

1 enables DEBUG MCP logs.

MARU_SEARCH_MAX_CONCURRENT

여러 페이지를 동시에 열 때 기본 개수(보통 5개).

Default fetch_bulk concurrency (5).
